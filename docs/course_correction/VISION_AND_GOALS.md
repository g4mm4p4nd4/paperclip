# Vision And Goals

## Vision

Paperclip is the governed control plane. Portfolio-OS is the deterministic truth plane. The combined system should repeatedly turn portfolio repos plus real market evidence into one frozen wedge, one shipped external proof artifact, and one measured distribution move without losing company boundaries or auditability.

## Non-Negotiables

- Closure over creation. No new routine or workflow is allowed without an explicit success condition and artifact.
- Portfolio-OS remains the canonical writer for frozen selections, snapshots, scaffolds, and dispatch artifacts.
- Paperclip owns orchestration, review, approvals, and company-scoped execution.
- Cross-company actions must use board-capable authority or fail loudly with a blocking artifact.
- Execution work is not done when code finishes. It is done after review, approval, and a durable artifact prove closure.
- Routine families must coalesce or pause under overlap. The default is not unbounded enqueue.

## Success Metrics For This Session

1. A current Portfolio-OS dispatch can be ingested by Paperclip without repeated schema errors.
2. A `ROUTE_TO_EXISTING_VENTURE` decision can create or update a real issue in the destination company and record the destination issue identifier back into Portfolio-OS artifacts.
3. Dispatch-created execution work enters a runtime-enforced path of executor -> QA review -> release approval instead of silently finishing.
4. The busiest LeadForge routine families have explicit concurrency and WIP controls with a concrete verification path.
5. At least one regression check exists that would have caught today's dispatch mismatch or cross-company routing failure before runtime.

## Daily Cadence

- Morning:
  - Inspect dispatch ingest health, routing blockers, and live routine overlap.
  - Confirm only one active execution issue exists per routine family.
- Midday:
  - Push one externally provable artifact toward shipped state.
  - Move any execution issue that is ready into QA review, not directly to done.
- End of day:
  - Reconcile routed venture work, pending approvals, and dispatch ledger state.
  - Record closure artifacts and stop orphaned or duplicate routine descendants.

## Weekly Cadence

- Reconcile the Paperclip routine roster against the intended orchestration model.
- Audit route-to-existing-venture outcomes for real destination issues and closure artifacts.
- Remove or pause routine families that violate WIP limits or generate duplicate descendants.
- Re-run regression checks against dispatch emission, ingest compatibility, and routing auth paths.
