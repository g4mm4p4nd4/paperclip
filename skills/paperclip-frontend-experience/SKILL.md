# Paperclip Frontend Experience

This skill exists to stop agents from shipping generic UI.

## When to Use

- landing pages and marketing surfaces
- hero sections and conversion flows
- forms that need trust, clarity, or gravitas
- dashboards and product UI that need visual hierarchy
- frontend work where interaction quality is part of the product value

## Core Principles

- Start from the message and user action, not from cards and containers.
- Make the primary action obvious in under five seconds.
- Treat typography, spacing, color, and motion as part of the product logic.
- Build responsive behavior intentionally, not as an afterthought.
- Keep loading, empty, success, and error states coherent with the main experience.

## Implementation Checklist

- establish a clear visual direction before coding
- choose purposeful hierarchy for headings, body copy, and CTAs
- ensure the interface works on mobile and desktop
- connect frontend states to real backend contracts, not fake placeholders
- preserve accessibility and keyboard behavior while adding motion or polish

## Quality Bar

- no generic placeholder hero copy
- no flat, lifeless default layouts when the page should persuade
- no form flow that hides validation, success, or failure
- no frontend implementation that ignores the backend data shape
